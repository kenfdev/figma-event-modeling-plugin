# F3.2: UpdateCustomFields

## Type
Command

## Description
User edits custom fields textarea for Command/Event/Query elements.

## User Story
As a user, I want to add custom field definitions to Command, Event, and Query elements to document their data structure.

## Acceptance Criteria
- [ ] Custom fields textarea shown for Command, Event, and Query elements
- [ ] Textarea accepts free-form text (e.g., `userId: string`, `amount: number`)
- [ ] No validation or structure enforcement on input
- [ ] Fields are stored in plugin data only (not visible outside plugin UI by default)
- [ ] Changes persist when element is deselected and reselected
- [ ] Actor elements do NOT show custom fields

## Dependencies
- F2.1: ViewSelectedElement
- F1.1, F1.2, F1.3: Command, Event, Query elements

## Technical Notes
- Input format is free-form text
- Storage: Plugin data only
- See F3.5 for visibility toggle
